1) What is File Handling?

File handling means your program can create, read, write, append, copy, move, and delete files on the computer. In C#, these features are provided mainly by the System.IO namespace.

Important concept: Your app does not ‚Äústore data inside C#‚Äù. It asks the OS to store data on disk using System.IO APIs.

2) When do we use files in real life?

Common real-world uses

  • Logs: Save application logs to a text file
  • Reports: Export report as CSV/TXT
  • Configuration: Read settings from JSON
  • Backups: Copy important files to a safe folder
  • Uploads: Save uploaded files (web apps)

Typical interview / assessment tasks

  • Read a text file and count words/frequency
  • Write results to an output file
  • Append logs with timestamps
  • Handle file-not-found & invalid paths

4) Paths + Working Directory (very important)

When you write code like "data.txt", it means ‚Äúdata.txt in the current working directory‚Äù. In exams and real apps, it is safer to use a full path or use AppContext.BaseDirectory.

Example: build a safe pathRecommended
using System;
using System.IO;

string baseDir = AppContext.BaseDirectory;
string filePath = Path.Combine(baseDir, "data.txt");

Console.WriteLine(filePath);
Common mistake: Using hard-coded paths like C:\temp\data.txt works only on your machine. Use Path.Combine and let Windows/Linux/Mac handle separators.

5) Read/Write Text Files (using File)

5.1 Write all text (create/overwrite)

using System;
using System.IO;

string filePath = Path.Combine(AppContext.BaseDirectory, "notes.txt");
File.WriteAllText(filePath, "Hello!\nThis is written by C#.\n");

Console.WriteLine("Written successfully.");

5.2 Read all text

using System;
using System.IO;

string filePath = Path.Combine(AppContext.BaseDirectory, "notes.txt");
string content = File.ReadAllText(filePath);

Console.WriteLine("File Content:");
Console.WriteLine(content);

5.3 Append text (add at end)

using System;
using System.IO;

string filePath = Path.Combine(AppContext.BaseDirectory, "notes.txt");
File.AppendAllText(filePath, $"Log at {DateTime.Now:HH:mm:ss}\n");

Console.WriteLine("Appended successfully.");
Exam tip: For small files, ReadAllText / WriteAllText is simplest. For large files, use StreamReader to read line-by-line.

6) Streams (StreamReader / StreamWriter)

Streams are used when you want more control and efficiency. Example: read file line-by-line without loading entire file into memory.

6.1 Write using StreamWriter (recommended pattern: using)

using System;
using System.IO;

string filePath = Path.Combine(AppContext.BaseDirectory, "biglog.txt");

using (var writer = new StreamWriter(filePath, append: true))
{
    writer.WriteLine("Line 1");
    writer.WriteLine("Line 2");
}

Console.WriteLine("StreamWriter done.");

6.2 Read line-by-line using StreamReader

using System;
using System.IO;

string filePath = Path.Combine(AppContext.BaseDirectory, "biglog.txt");

using (var reader = new StreamReader(filePath))
{
    string? line;
    while ((line = reader.ReadLine()) != null)
    {
        Console.WriteLine(line);
    }
}
Why ‚Äúusing‚Äù? It guarantees the file handle is closed even if an exception happens. Otherwise, the file may remain locked.

7) Copy / Move / Delete + Directory Operations

7.1 Copy a file

using System;
using System.IO;

string src = Path.Combine(AppContext.BaseDirectory, "notes.txt");
string dest = Path.Combine(AppContext.BaseDirectory, "backup-notes.txt");

File.Copy(src, dest, overwrite: true);
Console.WriteLine("Copied to backup.");

7.2 Move (rename) a file

using System;
using System.IO;

string oldPath = Path.Combine(AppContext.BaseDirectory, "backup-notes.txt");
string newPath = Path.Combine(AppContext.BaseDirectory, "backup-notes-renamed.txt");

File.Move(oldPath, newPath, overwrite: true);
Console.WriteLine("Moved (renamed).");

7.3 Delete a file (safe way)

using System;
using System.IO;

string filePath = Path.Combine(AppContext.BaseDirectory, "temp.txt");

if (File.Exists(filePath))
{
    File.Delete(filePath);
    Console.WriteLine("Deleted.");
}
else
{
    Console.WriteLine("File not found, nothing to delete.");
}

7.4 Create folder + list files

using System;
using System.IO;

string folder = Path.Combine(AppContext.BaseDirectory, "Reports");
Directory.CreateDirectory(folder);

foreach (var file in Directory.GetFiles(folder))
{
    Console.WriteLine(file);
}

8) Common Errors + Try/Catch (must know)

ErrorReasonFix
FileNotFoundException File path is wrong / file doesn‚Äôt exist Check with File.Exists
DirectoryNotFoundException Folder path doesn‚Äôt exist Create folder using Directory.CreateDirectory
UnauthorizedAccessException No permission to access path Use correct folder, run with permissions
IOException File is locked / in use / disk issues Close stream, retry, use ‚Äúusing‚Äù
Example: safe read with try/catchExam-ready
using System;
using System.IO;

try
{
    string path = Path.Combine(AppContext.BaseDirectory, "data.txt");

    if (!File.Exists(path))
    {
        Console.WriteLine("File not found.");
        return;
    }

    string text = File.ReadAllText(path);
    Console.WriteLine(text);
}
catch (UnauthorizedAccessException ex)
{
    Console.WriteLine("Permission issue: " + ex.Message);
}
catch (IOException ex)
{
    Console.WriteLine("File I/O issue: " + ex.Message);
}

9) Async File I/O (modern pattern)

Async file methods keep your app responsive (especially in GUI/web apps), and can improve throughput for I/O-heavy tasks. 

using System;
using System.IO;
using System.Threading.Tasks;

string path = Path.Combine(AppContext.BaseDirectory, "async.txt");
await File.WriteAllTextAsync(path, "Saved using async method!\n");
string read = await File.ReadAllTextAsync(path);

Console.WriteLine(read);
Exam hint: If your question says ‚Äúlarge files‚Äù or ‚Äúdo not block‚Äù, mention ReadAllTextAsync / StreamReader.ReadLineAsync.

10) Best Practices (Professional)

1) Always build paths using Path.Combine (avoid hard-coded separators).
2) Use using for streams/writers/readers to close handles safely.
3) Check File.Exists / Directory.Exists before read/delete.
4) Handle exceptions (FileNotFound, UnauthorizedAccess, IOException).
5) Prefer read-only folders for config and write into app data folders for logs.
6) For big files, read line-by-line (don‚Äôt load entire file).
