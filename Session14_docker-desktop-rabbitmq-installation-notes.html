Complete Notes: Docker Desktop Installation with RabbitMQ Installation for .NET

This guide explains how to install Docker Desktop on Windows, run RabbitMQ using Docker, access the RabbitMQ Management UI, test queues, and connect from a .NET application.

Best for: Freshers, .NET learners, backend developers, and anyone starting message queue practice using RabbitMQ.

1. Introduction

RabbitMQ is a message broker. It helps applications communicate with each other asynchronously. Instead of directly calling another service and waiting, one application can send a message to RabbitMQ, and another application can process it later.

Docker Desktop lets you run applications inside containers on Windows. Running RabbitMQ inside Docker is usually easier than doing a full native installation because you avoid local Erlang setup and can remove/recreate the broker quickly.

Simple idea: Your .NET application sends a message → RabbitMQ stores it in a queue → another .NET service or worker reads it and processes it.

2. Why use Docker + RabbitMQ

  • No need to manually install and manage RabbitMQ dependencies on your machine.
  • Easy to start, stop, remove, and recreate RabbitMQ.
  • Good for learning, testing, local development, and demos.
  • Works well with ASP.NET Core Web API, Worker Service, background jobs, and microservices.

Real-world .NET use cases

  • Order placed in an e-commerce app
  • Email notification processing
  • Invoice generation
  • Inventory update
  • Payment event processing
  • Log or audit event handling

3. Prerequisites

System requirements

  • Windows 10 or Windows 11
  • Administrator access for Docker Desktop installation
  • Virtualization enabled in BIOS/UEFI
  • WSL 2 enabled on Windows for Docker Desktop's recommended backend
  • Internet connection to pull Docker images
Important: RabbitMQ Docker image runs as a Linux container. Docker Desktop should be running in Linux containers mode.

Ports used by RabbitMQ

Port Purpose
5672 AMQP client communication from .NET or other applications
15672 RabbitMQ Management UI in browser

4. Install Docker Desktop on Windows

Step 1: Download Docker Desktop

Download Docker Desktop for Windows from the official Docker website.

Open the official Docker Desktop installation page and download the Windows installer.

Step 2: Run the installer

  1. Double-click the downloaded Docker Desktop installer.
  2. Proceed with the installation wizard.
  3. Keep WSL 2 based engine enabled if shown.
  4. Finish installation and restart your machine if prompted.

Step 3: Start Docker Desktop

  1. Open Docker Desktop from Start Menu.
  2. Wait until Docker shows that the engine is running.
  3. If prompted, allow required Windows features or WSL setup.

Step 4: Confirm Linux containers

If you previously worked with Windows containers, switch to Linux containers from the Docker tray menu.

Expected result: Docker Desktop opens successfully and shows the Docker engine as running.

5. Verify Docker Installation

Open Command Prompt, PowerShell, or Windows Terminal and run:

docker --version
docker version
docker info

You can also test Docker using a sample container:

docker run hello-world
If this works, Docker is installed and able to download and run containers correctly.

6. Run RabbitMQ in Docker

RabbitMQ one-line command

docker run -d --name rabbitmq -p 5672:5672 -p 15672:15672 rabbitmq:4-management

Command explanation

Part Meaning
-d Run container in detached mode (background)
--name rabbitmq Container name will be rabbitmq
-p 5672:5672 Expose AMQP port
-p 15672:15672 Expose Management UI port
rabbitmq:4-management RabbitMQ image with management plugin enabled

Check running containers

docker ps

See container logs

docker logs rabbitmq

Stop and start container

docker stop rabbitmq
docker start rabbitmq

Remove container

docker rm -f rabbitmq
If you get an error like no matching manifest for windows/amd64, Docker is likely in Windows containers mode. Switch to Linux containers and run the command again.

7. Open RabbitMQ Management UI

Open a browser and go to:

http://localhost:15672

Default credentials for a local setup are usually:

Username: guest
Password: guest

What you can do in the UI

  • View queues
  • Create exchanges
  • Publish messages
  • Get messages
  • Monitor connections and channels
  • See queue depth and message rates

8. Test RabbitMQ from the UI

Step 1: Create a queue

  1. Login to RabbitMQ Management UI.
  2. Open Queues and Streams.
  3. Click Add a new queue.
  4. Enter queue name: testqueue.
  5. Keep default settings and click Add queue.

Step 2: Publish a test message

  1. Open the testqueue.
  2. Go to Publish message.
  3. Enter payload such as:
{"message":"Hello RabbitMQ"}
  1. Click Publish message.

Step 3: Read the message

  1. In the same queue page, find Get messages.
  2. Click Get Message(s).
  3. You should see the message body.
Success check: If you can create a queue, publish a message, and retrieve it, RabbitMQ is working properly.

Optional: Run RabbitMQ with persistent data

If you want queue data to survive container deletion, mount a Docker volume:

docker run -d --name rabbitmq ^
  -p 5672:5672 ^
  -p 15672:15672 ^
  -v rabbitmq_data:/var/lib/rabbitmq ^
  rabbitmq:4-management

Use the above multi-line format in Command Prompt. In PowerShell or Git Bash, line continuation differs, so the one-line command is simpler for beginners.

10. Connect RabbitMQ with .NET

Create a console app

dotnet new console -n RabbitMqDemo
cd RabbitMqDemo
dotnet add package RabbitMQ.Client

Producer example

using RabbitMQ.Client;
using System.Text;

var factory = new ConnectionFactory()
{
    HostName = "localhost",
    UserName = "guest",
    Password = "guest"
};

await using var connection = await factory.CreateConnectionAsync();
await using var channel = await connection.CreateChannelAsync();

await channel.QueueDeclareAsync(
    queue: "testqueue",
    durable: false,
    exclusive: false,
    autoDelete: false,
    arguments: null);

var message = "Hello from .NET Producer";
var body = Encoding.UTF8.GetBytes(message);

await channel.BasicPublishAsync(
    exchange: string.Empty,
    routingKey: "testqueue",
    body: body);

Console.WriteLine("Message sent successfully.");

Consumer example

using RabbitMQ.Client;
using RabbitMQ.Client.Events;
using System.Text;

var factory = new ConnectionFactory()
{
    HostName = "localhost",
    UserName = "guest",
    Password = "guest"
};

await using var connection = await factory.CreateConnectionAsync();
await using var channel = await connection.CreateChannelAsync();

await channel.QueueDeclareAsync(
    queue: "testqueue",
    durable: false,
    exclusive: false,
    autoDelete: false,
    arguments: null);

var consumer = new AsyncEventingBasicConsumer(channel);

consumer.ReceivedAsync += async (model, ea) =>
{
    var body = ea.Body.ToArray();
    var message = Encoding.UTF8.GetString(body);
    Console.WriteLine($"Received: {message}");
    await Task.CompletedTask;
};

await channel.BasicConsumeAsync(
    queue: "testqueue",
    autoAck: true,
    consumer: consumer);

Console.WriteLine("Waiting for messages...");
Console.ReadLine();

How to test

  1. Run RabbitMQ container.
  2. Run the consumer app first.
  3. Run the producer app.
  4. Check console output in the consumer app.
  5. Also verify queue activity in RabbitMQ Management UI.
In a real ASP.NET Core project, the producer can be your Web API and the consumer can be a Worker Service using BackgroundService.

11. Troubleshooting

Problem: Docker Desktop not starting

  • Check whether WSL 2 is enabled.
  • Restart Docker Desktop.
  • Restart Windows.
  • Verify virtualization is enabled in BIOS.

Problem: RabbitMQ UI not opening

  • Check whether the container is running with docker ps.
  • Check logs using docker logs rabbitmq.
  • Confirm port 15672 is not blocked by another process.

Problem: Application cannot connect to RabbitMQ

  • Verify container is running.
  • Check host name is localhost.
  • Check port is 5672.
  • Check username/password.
  • Make sure the queue name matches exactly.

Problem: no matching manifest for windows/amd64

Switch Docker Desktop to Linux containers and run the RabbitMQ container again.

12. FAQ

Do I need Erlang when using RabbitMQ in Docker?

No. The Docker image already contains what RabbitMQ needs inside the container.

Should I use native RabbitMQ installation or Docker?

For learning and local .NET development, Docker is usually easier. For production and deeper OS-level management, teams may use Linux servers, Kubernetes, or managed environments.

Can I use RabbitMQ with ASP.NET Core Web API?

Yes. It is commonly used with Web API, Worker Services, microservices, background processing, and event-driven systems.

What should I learn next after basic setup?

  • Work Queues
  • Publish/Subscribe
  • Exchange types
  • Dead-letter queues
  • Retry mechanism
  • BackgroundService integration in ASP.NET Core
