NUnit in C#.NET – Detailed Beginner Notes with Examples

NUnit is a popular unit testing framework for C#.NET used to test individual units of code automatically.

1. Why Unit Testing?

  • Ensures code works as expected
  • Finds bugs early
  • Improves code quality
  • Makes refactoring safe

2. What is NUnit?

NUnit is an open-source testing framework inspired by JUnit. It allows developers to write and execute tests using attributes.

3. NUnit Architecture (Simple)

Your Code (Class / Method)
        ↓
NUnit Test Case
        ↓
Assertions (Expected vs Actual)
        ↓
Test Result (Pass / Fail)

5. Sample Coding Class (Business Logic)

Calculator.cs

using System;

namespace CalculatorApp
{
    public class Calculator
    {
        public int Add(int a, int b)
        {
            return a + b;
        }

        public int Subtract(int a, int b)
        {
            return a - b;
        }

        public int Multiply(int a, int b)
        {
            return a * b;
        }

        public int Divide(int a, int b)
        {
            if (b == 0)
                throw new DivideByZeroException("Divider cannot be zero");

            return a / b;
        }
    }
}
This class contains simple business logic that we will test using NUnit.

6. Creating NUnit Test Class

CalculatorTests.cs

using NUnit.Framework;
using CalculatorApp;
using System;

namespace CalculatorApp.Tests
{
    [TestFixture]
    public class CalculatorTests
    {
        private Calculator calculator;

        [SetUp]
        public void Setup()
        {
            calculator = new Calculator();
        }

        [Test]
        public void Add_WhenCalled_ReturnsCorrectSum()
        {
            int result = calculator.Add(10, 5);
            Assert.AreEqual(15, result);
        }

        [Test]
        public void Subtract_WhenCalled_ReturnsCorrectResult()
        {
            int result = calculator.Subtract(10, 3);
            Assert.AreEqual(7, result);
        }

        [Test]
        public void Multiply_WhenCalled_ReturnsCorrectResult()
        {
            int result = calculator.Multiply(4, 5);
            Assert.AreEqual(20, result);
        }

        [Test]
        public void Divide_ByZero_ThrowsException()
        {
            Assert.Throws(() => calculator.Divide(10, 0));
        }
    }
}

7. Important NUnit Attributes

AttributeDescription
[TestFixture]Marks a class as test class
[Test]Marks a test method
[SetUp]Runs before each test
[TearDown]Runs after each test
[TestCase]Parameterized tests

8. Parameterized Test Example

[TestCase(2, 3, 5)]
[TestCase(5, 5, 10)]
[TestCase(-1, 1, 0)]
public void Add_WithMultipleInputs_ReturnsCorrectSum(int a, int b, int expected)
{
    int result = calculator.Add(a, b);
    Assert.AreEqual(expected, result);
}

9. Common Assertions

AssertionPurpose
Assert.AreEqualCheck equality
Assert.IsTrueCheck true condition
Assert.IsFalseCheck false condition
Assert.ThrowsCheck exceptions
Assert.NotNullCheck object not null

10. Test Execution Flow

[TestFixture] Loaded
    ↓
[SetUp] runs
    ↓
[Test] executes
    ↓
Assert validates result
    ↓
Pass / Fail

11. Best Practices

  • One assertion per test (recommended)
  • Test method names should be descriptive
  • Do not write logic inside test
  • Keep tests independent
  • Test positive and negative scenarios
Avoid: Testing private methods directly. Test through public methods.

12. Summary

  • NUnit is a powerful testing framework for C#
  • Separates business logic and testing
  • Improves reliability and maintainability
  • Essential skill for .NET developers
